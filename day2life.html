Analytics for Developers

By using MySQL database as the backend, I connected to the server to add datasets and graph the datasets. I accessed databases based Item and used HTML/CSS in order to create pages for the Item adding forms, Item view tables, and Item detail pages. After creating web pages, I used MySQL queries to extract information on daily and weekly intervals. Along with those queries, I used JQuery to create generate charts and graphs for the clients to easily view the datasets on their demands. Since the company is only partnering with Korean clients, all the screenshots are in Korean.

Item List and Item Review List

Before jumping straight into backend, I started by understanding the architecture of Model-View-Controller. Since I had to use Spring Boot, Eclipse, Java, and ThymeLeaf, I spent a few days learning about the frameworks and the programs. After grasping the knowledge, I moved onto the clients page and the SQL database. I specifically worked on the item columns from the side navigation bar. I had to begin with creating an Item Controller in the server that connects all models and viewers. After the controller, I created models and viewers based off the SQL database. The first page that I worked on was an overall item listview with the columns pulled straight from the SQL database and a button that leads to the list of reviews for the item. After working on the item listview, I coded the item review list, which was a very similar process as item listview but separate viewer.

Item Detail and Item Create Pages

After creating the list pages, I moved onto working on the detail pages, which are pages where users can edit and create items as clients. Item create page and item edit page have very similar structure, and thus the work was iterative. There were six types of input: text, number, blocked, drop-down, date, and file upload.

Data Visualization for Recommended Events

Based on the list they’ve already had, I had to create an analytics page where clients can view the statistics and a graph of the statistics. Whether the client chooses to view in daily or weekly intervals, the graph changes the segments. I used Chart.js and MySQL functions to create graphs and connect the tables with the database. The user can deselect any category from the five categories and the graph will modify its default view depending on those selected categories.
